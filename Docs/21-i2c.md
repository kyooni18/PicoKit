# PicoKit Documentation

## Chapter 21: I2C


Create I2C0 or I2C1:

```swift
let i2c = try PicoI2C(
    .i2c0,
    frequency: .kilohertz(400),
    sda: PicoPin(4),
    scl: PicoPin(5)
)
```

Write bytes:

```swift
let count = try i2c.write(
    address: 0x3C,
    bytes: [0x00, 0xAF],
    timeout: .milliseconds(100)
)
```

Read bytes:

```swift
let bytes = try i2c.read(
    address: 0x3C,
    count: 4,
    timeout: .milliseconds(100)
)
```

PicoKit accepts normal 7-bit peripheral addresses from `0x08` through `0x77`. Reserved addresses outside that range are rejected.

The current API performs complete standalone read or write transactions with `nostop` disabled. Combined repeated-start register transactions are not directly represented by the public API.

# PicoKit documentation

The documentation is intentionally kept to four maintained guides. Start with
the first one; use the API reference only when you need an exact declaration.

1. [Getting started](getting-started.md) — boards, setup, project creation,
   build, flash, monitor, and a complete firmware entry point.
2. [Hardware guide](hardware-guide.md) — sketch API, low-level API, GPIO,
   serial, PWM, ADC, I2C, SPI, interrupts, watchdog, testing, and limits.
3. [API reference](api-reference.md) — complete public surface.
4. [Integration and performance](integration.md) — C/C++ libraries,
   migration, DMA, and benchmarking.

The repository [README](../README.md) is the short introduction and links to
these guides.
